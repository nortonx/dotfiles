# Purpose

Generate a report to leadership based on provided instructions and data.

## Usage

/commandname [arguments]

## Tasks

1. The report should have a header with the date and time of generation.
2. The report should include a summary of the data provided.
3. The report should be formatted in a professional manner.
4. The report should cover Pros and Cons, impact, benefits, quick-wins, and recommendations.
5. The list of items, if applicable, should be organized showing what is critical,
whatneeds to be improved, and what is already in good shape.
6. The list of items should be sorted in level of severity and will be represented
with colored circles. Red for critical, yellow for improvement, and green for
what is ingood shape.
7. The report should remove duplicated entries.
8. The authors will always be 'The Frontend Team'

## Output

Tone Guidelines

- Factual and professional
- Non-alarmist
- Focus on process improvement, not blame
- Suitable for leadership audience

Tooling

- If a tool is mentioned, use context7 to learn about documentation and best practices.
- Use the command `automate-this` to set initial parameters.
- Always check current date and time to avoid knowledge cutoff.

## Notes

Always consider best practices and follow the tone guidelines.
